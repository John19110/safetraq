About SafeTraq

Turning every smartphone into a safety companion

SafeTraq is a graduation project that combines mobile sensor fusion, on-device AI, and cloud-backed emergency workflows into one cohesive driving safety platform.

The Problem

Road accidents claim over a million lives every year. The critical window after a crash is often wasted before emergency services or family are alerted—especially when victims cannot call for help.

The Motivation

Smartphones already carry the sensors needed to detect severe impacts. SafeTraq was built to turn that hardware into a life-saving safety net—without requiring dedicated telematics devices.

The Solution

On-device machine learning analyzes motion in real time. When a crash is confirmed, SafeTraq triggers voice alerts, vibration, a countdown, and automatic SOS SMS with GPS coordinates.

The Impact

By shortening the time between collision and contact with help, SafeTraq aims to improve outcomes for drivers, passengers, and the people who care about them.

Features

Everything needed for intelligent driving safety

From sensor fusion to emergency SMS—SafeTraq delivers a complete crash detection and response pipeline on Android.

Automatic Crash Detection

On-device TensorFlow Lite models analyze accelerometer, gyroscope, and speed data to detect potential crashes in real time—no cloud inference required.

Driving Mode

One tap activates a foreground monitoring service that keeps crash detection running even when the screen is locked or the app is backgrounded.

Live Speed Monitoring

Google Play Services fused location delivers GPS speed as a core ML feature and contextual gate to reduce false positives while stationary.

Accelerometer Monitoring

High-frequency accelerometer sampling captures impact signatures with dedicated sensor threading for reliable background collection.

Gyroscope Monitoring

Devices with gyroscopes use a CNN+GRU model with full 6-axis motion data; others automatically fall back to an accelerometer+speed classifier.

Intelligent Sensor Fusion

60-sample sliding windows combine preprocessed accel, gyro, and speed into a unified feature vector for ML inference every 50 ms.

False Positive Reduction

Triple confirmation requires model confidence ≥95%, peak impact ≥24 m/s², and three consecutive qualifying frames before triggering an alert.

Emergency Contact Alerts

Configure up to two emergency contacts with relationship roles. They receive SOS messages automatically when a crash is confirmed.

Live GPS Sharing

Every SOS message includes a Google Maps link with the user's coordinates, fetched with timeout-safe location logic.

SOS System

Manual SOS button and automatic post-countdown dispatch send multipart SMS to all emergency contacts via a dedicated foreground service.

Local Emergency Numbers

In-app emergency guide with one-tap speed dials for ambulance, police, fire, and highway services plus post-crash action steps.

Route & POI Intelligence

After a crash, SafeTraq surfaces nearest traffic departments using bundled CSV data and optional OpenRouteService road-distance ranking.

Secure Authentication

Firebase Authentication with email verification ensures only verified users access monitoring, profiles, and emergency features.

User Profiles

Editable profile with name, phone, and emergency contacts synced to Cloud Firestore with real-time listeners.

Crash History

Every detected crash is geotagged and stored in Firestore with a live crash counter on the account screen.

Background Monitoring

DrivingMonitorService runs as a foreground service with sensor listeners on a dedicated HandlerThread for reliable background operation.

Firebase Integration

Cloud Firestore stores profiles, emergency contacts, and crash records. Firebase Auth handles secure sign-in and account lifecycle.

Crash Detection Logic

How SafeTraq decides a crash happened

Multi-layered analysis combines ML inference with physics-based gates and human confirmation.

1

Speed Analysis

GPS speed feeds the ML model and a driving-context weight—crashes are suppressed when stationary.

2

Accelerometer Analysis

3-axis acceleration is preprocessed with smoothing, calibration, and peak linear-accel tracking.

3

Gyroscope Analysis

Rotational rates complete the 7-feature vector on gyro-equipped devices.

4

Sensor Fusion

60-sample windows at 50 ms intervals feed the TFLite interpreter for crash probability.

5

Threshold Detection

Raw probability must exceed 0.95 with impact ≥24 m/s² for three consecutive frames.

6

Countdown Verification

10-second user confirmation prevents false alarms before SOS is dispatched.

7

False Positive Prevention

Stationary detection, EMA smoothing, and speed/impact context weights filter benign events.
